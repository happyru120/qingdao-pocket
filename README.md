# 칭다오 포켓

칭다오 여행 현장에서 빠르게 결정하기 위한 **개인용 모바일 여행 노트**입니다.

장소를 많이 보여주는 여행 포털보다 아래 질문에 빠르게 답하는 데 초점을 맞췄습니다.

- 지금 어디를 갈 수 있는지
- 이 식당에서 두 명이 뭘 얼마나 주문해야 하는지
- 정확한 중국어 상호와 고덕지도 검색 링크가 무엇인지
- 직원에게 바로 들려줄 중국어 문장이 무엇인지
- 마트에서 어떤 상품을 사야 하는지

## 현재 구현된 화면

- 홈
- 장소 검색 및 카테고리 필터
- 장소 상세와 2인 주문 가이드
- 영업시간 검증 상태
- 고덕지도 URI 연결
- 여행 일정 초안
- 중국어 검색·음성·복사·큰 글씨 보기
- 기념품 체크리스트
- 저장한 장소
- 2026 칭다오 국제맥주축제 전용 페이지

## 실행

```bash
npm install
npm run dev
```

브라우저에서 표시되는 주소를 휴대폰으로 열면 모바일 UI를 확인할 수 있습니다. Mac에서는 `start.command`를 실행해도 됩니다.

## 프로덕션 빌드

```bash
npm run build
npm run preview
```

## 디자인 원칙

- 모바일 360–430px 우선
- UI 이모지 미사용
- `lucide-react` 아이콘 통일
- 점과 텍스트만 있는 상태 배지 남발 금지
- 카드 안에 카드를 반복하지 않기
- 모든 정보에 임의 점수를 붙이지 않기
- 영업시간·가격·지점은 검증 상태와 마지막 확인일 표시
- 실제 현장에서 행동으로 이어지는 팁을 우선

## 데이터 상태

현재 장소 데이터는 대화에서 수집한 후보를 바탕으로 만든 **MVP 초안**입니다.

- `확인됨`: 출처 또는 사용자가 제공한 구체 정보가 있음
- `부분 확인`: 상호·주소 등 일부만 확인됨
- `재확인 필요`: 영업시간·가격·지점 등을 방문 전에 다시 확인해야 함

정확하지 않은 정보는 빈칸을 억지로 채우지 않고 재확인 상태로 남깁니다.

## 주요 폴더

```text
src/
  components/   공통 UI
  context/      저장·쇼핑 체크 상태
  data/         장소·회화·쇼핑·일정 데이터
  lib/          고덕지도 링크와 영업 상태 계산
  pages/        화면 단위 컴포넌트
```

## 다음 개발 순서

1. 숙소의 정확한 고덕 POI와 좌표 확정
2. 각 장소의 정확한 지점·영업시간·가격 재검증
3. 식당별 2인·3인 주문 조합 보강
4. 메이투안 검색용 중국어 상호와 배달 후보 정리
5. 실제 휴대폰에서 터치·스크롤·음성 테스트
6. Vercel 또는 개인 서버 배포

## GitHub Pages 배포

이 저장소에는 `.github/workflows/deploy-pages.yml`이 포함되어 있습니다.

1. GitHub에 새 저장소를 만들고 소스를 `main` 브랜치에 푸시합니다.
2. 저장소의 **Settings → Pages**로 이동합니다.
3. **Build and deployment → Source**를 **GitHub Actions**로 선택합니다.
4. 이후 `main`에 푸시할 때마다 자동으로 빌드·배포됩니다.

프로젝트 저장소 이름이 `qingdao-pocket`이라면 기본 주소는 아래 형태입니다.

```text
https://<github-id>.github.io/qingdao-pocket/
```

> 주의: GitHub Pages 사이트는 인터넷에 공개됩니다. 숙소명·여행 날짜 등 개인 정보가 들어간 상태로 배포할 때는 공개 범위를 먼저 확인하세요.
